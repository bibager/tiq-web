2

Install the MCP server

Run this command in your terminal. It registers TrackIQ with Claude Desktop.

# Add TrackIQ to Claude Desktop
$ claude mcp add trackiq "https://trackiq.com/mcp"
# Restart Claude Desktop, then verify
$ claude mcp list
→ trackiq · connected · 16 tools
Or edit ~/Library/Application Support/Claude/claude_desktop_config.json manually. Show me how →
3

Connect your Amazon account

One-click OAuth through Amazon Seller Central. We never see your password — Amazon issues a scoped token directly to TrackIQ.

amazon sellercentral.amazon.com / oauth

TrackIQ wants permission to access your account

By authorizing, you'll let TrackIQ read the following data on your behalf:

  • Sales & Orders Reports
  • Advertising Performance
  • Inventory & Listings
  • Brand Analytics (Plus tier)

You can revoke this access at any time from Seller Central → Apps & Services.

Hit a snag?

Troubleshooting.

"Claude doesn't see TrackIQ"

Restart Claude Desktop after running the install command. Check claude mcp list; if TrackIQ isn't there, run install again with the verbose flag.

"OAuth says invalid scope"

Make sure you're signed into Seller Central as an Admin user. Marketplace-restricted accounts can't grant the Brand Analytics scope.

"Tool call returned empty"

If your data is fewer than 24 hours old, Amazon's reporting API may not have it yet. Try a date range from 2+ days ago.
